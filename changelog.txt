# 4.0

## Features
- angular 9 compatible
- new global options: `disableAutoID` (boolean) and `timestampObject` (boolean)
- parse saveAll
```ts
import { object } from '@reative/parse';
import { collection } from '@reative/core';

 collection(`Activity`)
      .driver(`parse`)
      .set(
        [
          object(`Activity`, { message: 'hello' }),
          object(`Activity`, { message: 'there' })
        ],
        { all: true }
      )
      .subscribe(
        r => console.log(`success`, r),
        err => console.log(`err`, err)
      );
```

## Deprecations

- .http(fn) method doesnt work inside the constructor of instances generated by @Collection decorator. instead add .http(fn) in every chaining call
- http is the default driver now since firebase/firestore/parse lives on their respective packages. pay attention to specify a global or local driver to the call.
- `RecordsModule` has been replaced to `ReativeModule`
- now every server call should respond in the same format as browser unless you set .raw(true)
```ts
// server.js - server call working just as client
collection(`Todo`)
  .findOne()
  .toPromise()
  .then(todo=>console.log(todo))
  
// if you need raw data
collection(`Todo`)
  .raw(true)
  .findOne()
  .toPromise()
  .then(response=>console.log(response))
```
- the global timestamp option has been changed. replace `timestamp: false` to `disableTimestamp: true`



## A typical parse configuration

```ts
import { ReativeModule } from '@reative/angular';

ReativeModule.forRoot({
   silent: true,
   driver: 'parse',
   identifier: 'objectId',
   disableAutoID: true,
   disableTimestamp: true
 })
 ```
